# Platform owns the system image; the user owns their home

The replaceable system volume — base OS, common developer tooling, and agent binaries pinned at AMI build time with auto-update disabled — is exclusively platform-owned and refreshed by rebuilding the AMI on a weekly cadence. A stopped Environment adopts a newer promoted AMI silently at its next start (the start is fulfilled as a Runtime replacement); upgrades never interrupt a running session and require no user action. Everything the user installs or configures lives in the durable data volume (home, workspace, services, cache), and the image preconfigures user-space package managers to install there by default; system-path customizations (`sudo apt …`) are ephemeral by contract and vanish on replacement. A deterministic, replicable per-user toolset (Nix-style) is a recorded open decision, not an MVP commitment (2026-07-18).
